# Documentacion del SaaS para negocios de belleza, unas y manicure

Fecha: 04/05/2026

## 1. Resumen ejecutivo

La idea recomendada es crear un SaaS vertical para manicuristas, estudios de unas, salones de belleza pequenos, emprendedoras independientes y negocios de estetica en Peru.

El producto principal no debe empezar como marketplace puro. Debe empezar como software de gestion para que cada negocio pueda ordenar su operacion diaria, vender mejor, controlar su agenda, registrar pagos, administrar clientas, controlar inventario y entender si realmente esta ganando dinero.

El marketplace puede ser una segunda etapa, cuando ya existan suficientes negocios usando el sistema. Primero se construye valor para el negocio; luego se agrega visibilidad para conseguir nuevas clientas.

Propuesta central:

> El sistema que ayuda a manicuristas y salones de belleza a llenar su agenda, reducir plantones, controlar caja e inventario, y hacer volver a sus clientas.

## 2. Contexto del mercado peruano

El rubro oficial mas cercano es "Salones de belleza", dentro de actividades de belleza y tratamientos personales. En Peru no existe una clasificacion oficial separada exclusivamente para "unas" o "manicure"; normalmente estos negocios aparecen agrupados como salones de belleza, peluqueria, estetica o tratamientos personales.

Hallazgos relevantes de la investigacion previa:

- INEI reporto 49,925 empresas activas de salones de belleza en 2024.
- Con altas y bajas de 2025, el stock operativo estimado se aproxima a 53,742 negocios formales o semi-formales.
- El rubro esta muy atomizado: la mayoria son microempresas o personas naturales.
- La conduccion femenina es muy alta en salones de belleza.
- El canal comercial dominante es WhatsApp, Instagram, recomendaciones y agenda manual.
- Muchas emprendedoras no tienen control real de costos, inventario, recompra, historial de clientas ni margen por servicio.

Implicancia:

Hay oportunidad para un producto simple, movil, economico y especializado. No debe parecer un ERP pesado. Debe parecer una herramienta diaria para vender, organizarse y ganar mas.

## 3. Usuarios objetivo

### 3.1 Manicurista independiente

Trabaja sola, atiende en casa, a domicilio o en un espacio pequeno. Usa WhatsApp, Instagram, cuaderno o notas del celular.

Necesita:

- Agenda simple.
- Recordatorios.
- Registro de pagos por Yape, Plin y efectivo.
- Historial de clientas.
- Control basico de insumos.
- Saber cuanto gana al mes.

### 3.2 Studio pequeno de unas

Tiene 2 a 5 tecnicas, varias estaciones de trabajo y agenda compartida.

Necesita:

- Agenda por tecnica.
- Control de comisiones.
- Caja diaria.
- Inventario por insumo.
- Servicios con precios y duracion.
- Reportes por tecnica, servicio y metodo de pago.

### 3.3 Salon de belleza integral

Ofrece unas, cabello, cejas, pestanas, depilacion, maquillaje u otros servicios.

Necesita:

- Agenda por area o especialista.
- Catalogo amplio de servicios.
- Inventario mas completo.
- Roles de usuario.
- Comisiones.
- Reportes.
- Productos para venta.

### 3.4 Emprendedora que quiere formalizarse

Quiere ordenar su negocio, emitir mejores reportes para su contador y empezar a tomar decisiones con datos.

Necesita:

- Registro de ventas.
- Exportacion de datos.
- Control de ingresos y gastos.
- Reportes mensuales.
- Ficha de clientas.

## 4. Modelo recomendado

### 4.1 No empezar como marketplace puro

Un marketplace necesita dos lados al mismo tiempo:

- Muchos negocios registrados.
- Muchas clientas buscando servicios.

Eso es dificil al inicio porque si no hay oferta, las clientas no vuelven; si no hay demanda, los salones no pagan.

### 4.2 Empezar como SaaS de gestion

El primer producto debe resolver problemas internos del negocio:

- Agenda desordenada.
- Clientas que no llegan.
- Falta de control de pagos.
- Falta de historial.
- Falta de inventario.
- No saber que servicio deja mas margen.
- No tener base de datos de clientas.
- No tener una pagina simple para reservar.

### 4.3 Luego agregar capa marketplace

Cuando existan suficientes negocios en la plataforma, se puede activar un directorio:

- Buscar manicure cerca.
- Filtrar por distrito.
- Filtrar por disponibilidad.
- Ver precios.
- Ver fotos.
- Reservar.
- Contactar por WhatsApp.

El marketplace puede monetizarse con:

- Posicionamiento destacado.
- Comision por reserva.
- Publicidad local.
- Plan premium.
- Campanas para captar clientas.

## 7. Modulos del sistema final

### 7.1 Dashboard principal

Debe mostrar rapidamente:

- Citas de hoy.
- Ventas del dia.

- Servicios mas vendidos.

### 7.2 Agenda

Funciones:

- Citas por tecnica.
- Estados: pendiente, confirmada, atendida, cancelada, no asistio.
- Duracion por servicio.
- Bloqueo de horarios.
- Reprogramacion rapida.
- Comentarios internos.
- Anticipo asociado.
- Metodo de pago.
- Servicio realizado.

Estados sugeridos:

- Solicitada.
- Confirmada.
- En atencion.
- Finalizada.
- Cancelada.
- No asistio.

### 7.3 Reservas online

Cada negocio debe tener una pagina publica:

`tuapp.com/studio-bella`

La pagina debe mostrar:

- Nombre del negocio.
- Fotos.
- Servicios.
- Precios.
- Duracion.
- Ubicacion.
- Horarios.
- Boton reservar.
- Boton WhatsApp.
- Politicas de anticipo o cancelacion.

La reserva puede funcionar asi:

1. Clienta elige servicio.
2. Elige tecnica, si aplica.
3. Elige horario disponible.
4. Ingresa nombre y telefono.
5. El sistema crea cita pendiente.
6. Se abre WhatsApp con mensaje prellenado para confirmar.
7. La duena confirma manualmente.

### 7.6 Servicios

Cada servicio debe tener:

- Nombre.
- Categoria.
- Precio.
- Duracion.
- Descripcion corta.
- Imagen.
- Tecnicas que lo realizan.
- Insumos asociados.
- Comision asociada.
- Estado activo/inactivo.

Ejemplos:

- Manicure clasica.
- Manicure gel.
- Rubber gel.
- Soft gel.
- Acrilicas.
- Esculpidas.
- Pedicure spa.
- Retiro de gel.
- Diseno adicional.
- Reparacion de una.

### 7.7 Comisiones

Modelos de comision:

- Porcentaje por servicio.
- Monto fijo por servicio.
- Comision por tecnica.
- Comision mixta.
- Bono por meta.

Ejemplo:

- Servicio: acrilicas.
- Precio: S/120.
- Comision tecnica: 40%.
- Comision: S/48.
- Material estimado: S/12.
- Margen antes de otros gastos: S/60.

### 7.8 Inventario

El inventario debe ser uno de los diferenciales fuertes del SaaS. Muchas manicuristas compran insumos, pero no saben cuanto gastan ni cuanto consume cada servicio.

Tipos de inventario:

| Tipo | Ejemplos |
|---|---|
| Insumos consumibles | Gel, base, top, monomero, acrilico, esmaltes, removedor, alcohol |
| Descartables | Limas, guantes, palitos, algodon, toallas, mascarillas |
| Productos de venta | Aceite de cuticula, cremas, tratamientos, press-on nails |
| Herramientas/equipos | Lamparas, torno, brocas, sillas, esterilizador |

Campos por producto:

- Nombre.
- Categoria.
- Marca.
- Unidad de compra.
- Contenido.
- Stock actual.
- Stock minimo.
- Costo de compra.
- Costo unitario calculado.
- Proveedor.
- Fecha de compra.
- Fecha de vencimiento.
- Ubicacion.
- Foto.
- Estado.

Ejemplo de producto:

- Nombre: Base Rubber Gel.
- Categoria: Base gel.
- Marca: X.
- Unidad de compra: frasco.
- Contenido: 15 ml.
- Costo de compra: S/35.
- Costo por ml: S/2.33.
- Stock minimo: 3 ml.

### 7.9 Recetas por servicio

Esta es la parte mas valiosa del inventario.

Cada servicio debe tener una receta de consumo estimado. Cuando una cita se marca como finalizada, el sistema descuenta automaticamente esos insumos.

Ejemplo: Manicure gel

| Insumo | Consumo estimado | Costo estimado |
|---|---:|---:|
| Base gel | 0.4 ml | S/0.90 |
| Color gel | 0.8 ml | S/1.60 |
| Top coat | 0.3 ml | S/0.70 |
| Lima | 1 unidad | S/0.30 |
| Algodon/toallitas | 5 unidades | S/0.40 |
| Cleaner/alcohol | 3 ml | S/0.25 |
| Total material |  | S/4.15 |

Si el servicio se vende a S/60, el sistema puede mostrar:

- Precio: S/60.
- Materiales: S/4.15.
- Comision: S/24, si aplica 40%.
- Margen estimado: S/31.85.

Esto ayuda a la duena a entender si sus precios estan bien.

### 7.12 Alertas de inventario

Alertas utiles:

### 7.13 Marketing y fidelizacion

Sin API oficial de WhatsApp, el sistema debe trabajar con mensajes asistidos.

Funciones:

- Plantillas prellenadas.
- Boton abrir WhatsApp.
- Lista de clientas para contactar.
- Recordatorio manual.
- Campanas simples.
- Cumpleanos.
- Retoque.
- Clienta inactiva.
- Referidos.
- Tarjeta de sellos digital.

Ejemplos de plantillas:

Confirmacion:

> Hola {nombre}, tu cita en {negocio} esta confirmada para el {fecha} a las {hora}. Servicio: {servicio}. Te esperamos.

Recordatorio:

> Hola {nombre}, te recordamos tu cita manana a las {hora}. Si necesitas reprogramar, avisanos con anticipacion.

Retoque:

> Hola {nombre}, ya pasaron {dias} dias desde tu ultimo servicio. Si deseas retoque, tenemos horarios disponibles esta semana.

Cumpleanos:

> Feliz cumpleanos {nombre}. En {negocio} queremos regalarte un beneficio especial para tu proxima visita.

### 7.14 WhatsApp sin API oficial

Opciones permitidas para la primera etapa:

- Link `wa.me`.
- Mensajes prellenados.
- Boton "copiar mensaje".
- Boton "abrir WhatsApp".
- QR de WhatsApp.
- Uso de WhatsApp Business App.
- Plantillas internas.

No recomendado:

- Automatizar WhatsApp Web de forma no oficial.
- Usar bots no autorizados.
- Enviar mensajes masivos sin consentimiento.
- Depender de extensiones externas.

WhatsApp Business App ya permite perfil, catalogo, respuestas rapidas, etiquetas, mensajes de ausencia, enlaces directos y codigos QR. Tu SaaS debe complementar eso, no pelearse con eso.

- Telefonos.

- Fotos.
- Historial de servicios.

- Tener politica de privacidad.
- Pedir consentimiento para comunicaciones promocionales.

- No compartir datos de clientas entre negocios.
- Registrar quien accede a informacion sensible.

## 18. Flujo principal de uso

### Flujo de reserva

1. Clienta entra a pagina publica.
2. Elige servicio.
3. Elige fecha y hora.
4. Deja nombre y telefono.
5. Se crea cita pendiente.
6. El sistema genera mensaje para WhatsApp.
7. La duena confirma.
8. La cita aparece en agenda.

### Flujo de atencion

1. Llega la clienta.
2. Se marca cita en atencion.
3. Se finaliza el servicio.
4. Se registra pago.
5. Se descuenta inventario segun receta.
6. Se actualiza historial de clienta.
7. Se agenda sugerencia de retoque.

### Flujo de inventario

1. Se registra compra.
2. Aumenta stock.
3. Se atienden servicios.
4. Disminuye stock por receta.
5. El sistema alerta stock bajo.
6. Se genera lista de compras sugerida.

## 20. Riesgos y como reducirlos

Riesgo: las emprendedoras no quieren pagar software.

Solucion:

- Precio bajo de entrada.
- Prueba gratis.
- Onboarding facil.
- Mostrar ahorro y ventas recuperadas.

Riesgo: el inventario parece complicado.

Solucion:

- Empezar con plantillas.
- Permitir inventario basico primero.
- Recetas prearmadas.
- Ajuste manual facil.

Riesgo: esperan mensajes automaticos por WhatsApp.

Solucion:

- Explicar que la primera version usa WhatsApp asistido.
- Botones rapidos.
- Plantillas.
- Integracion oficial en planes futuros.

Riesgo: marketplace sin usuarios.

Solucion:

- No empezar por marketplace.
- Primero SaaS y paginas publicas.
- Activar directorio cuando haya masa critica.
